À propos de ce projet

Ce projet a été réalisé dans le cadre d’un BUT Informatique, première année. Il consiste à développer une plateforme web permettant de suivre la qualité de l’eau en France grâce aux données publiques issues de la toile Eaufrance.

Objectifs du projet

Exploiter un jeu de données parmi les 13 API disponibles pour les services de surveillance de l’eau. Stocker localement les données statiques (informations sur les points de mesure). Interroger en temps réel l’API Hub’eau pour récupérer les données dynamiques (qualité de l’eau). Offrir aux utilisateurs une interface simple pour consulter ces données.

Compétences développées

Conception et modélisation d’une base de données relationnelle à partir d’un cahier des charges. Création et gestion d’une base de données locale pour stocker les données statiques. Développement d’une application web avec Python et Flask. Implémentation d’une architecture MVC pour séparer modèle, vue et contrôleur. Interaction avec une API pour récupérer des données dynamiques. Développement d’une interface utilisateur claire, avec affichage des données et navigation simple. Tests et validation des fonctionnalités.

Fonctionnalités principales

Importation et mise à jour des données statiques via un script automatisé. Consultation des points de mesure (localisation, type, description). Affichage des indicateurs de qualité de l’eau récupérés en temps réel. Filtrage des données par zone ou point de mesure. Préparation à l’intégration de cartes interactives et graphiques. Gestion des erreurs et robustesse face aux indisponibilités de l’API.
